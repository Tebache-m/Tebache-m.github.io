Contraintes générales

Cadre universitaire

Le projet est développé dans un cadre académique imposant certaines limites techniques et fonctionnelles.

Ces contraintes garantissent la faisabilité et la cohérence du développement.

Contraintes techniques

Environnement logiciel

  • Application graphique en 2D
  • Vue du dessus
  • Affichage sur un écran unique
  • Absence de défilement
  • Performances optimisées

Contraintes spatiales

Organisation de la carte

L’espace de jeu est volontairement limité afin de favoriser une gestion stratégique.

  • Carte fixe
  • Emplacements prédéfinis
  • Zone entièrement visible
  • Déblocage progressif

Contraintes de conception

Architecture modulaire

Le projet adopte une structure modulaire permettant l’ajout de nouvelles fonctionnalités sans refonte majeure.

  • Modules indépendants
  • Code structuré
  • Maintenance facilitée
  • Évolutivité
